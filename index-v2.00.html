HeadJS, a small library for Responsive Design, Feature Detections & Ressource Loading

When you need a quick cross-browser compatible hand, don't bang your head. HeadJS is here to help you out !

  • Load JS & CSS asyncronously and in parallel
  • Load one asset if a condition is met, or fallback to another one
  • Manage script dependencies, and execute callbacks once they are loaded
  • Use cross-browser compatible CSS to code for different resolutions & devices
  • Fix quirks in specific browsers by quickly applying dedicated CSS automatically
  • Detect browsers & their versions
  • Check if a client supports certain Browser, HTML5 and CSS3 features
  • Generate JS and CSS classes automatically for detected browsers & features
  • Know if a user is in landscape or portrait mode
  • Know if a user is on a mobile or desktop device
  • Make incompatible browsers HTML5 compatible
  • ....

Your imagination, Your limit

Responsive design (Think @media-queries, and more)

Achieve responsive design with CSS that targets different screen resolutions, paths, states and browsers.

More than media-queries, and compatible in all browsers.

{% highlight css %} /* So how hard is it to write this in CSS ? */ if (head.page.index && head.browser.ie && head.browser.version < 8 && head.screen.innerHeight < 800) { $(".profile-list").show(0); } /* Not very hard */ #page-index.ie8.h-lt800 .profile-list { display: none; } {% endhighlight %}

And like you just saw above, anything you can write with HeadJS in CSS you can also write it in JavaScript !

mobile, desktop, phone, tablet, landscape, portrait, touch, viewport height, viewport width, screen height, screen width, browser height, browser width, browser name (ie, chrome, ff, ios, android, webkit, opera), browser version, current folder, current page, current hashtag

Ressource Loading (Direct or Conditional)

Manage your library's dependencies. Load javascript and stylesheets in parallel but execute them in the correct order.

On DocumentReady, if the user is using IE8, you need to load 2 scripts, fire a callback, and then apply a stylesheet ? Yeah right..

Easy.

{% highlight css %} head.ready(function() { head.test(head.browser.ie, function() { head.load(["file1.js", "file2.js"], function() { // do stuff ... head.load("custom.style.css"); }); }, null }); }); {% endhighlight %}

Of course you can use each of those functions (ready, test, load) individually too.


Feature Detection (Like Modernizr, with extras)

Detect various browsers and their features. Target HTML5 and CSS3 safely.

What about device & browser features, or custom detections ?

{% highlight css %} /* does the client have localStorage ? */ if (head.features.localStorage) { // do stuff } /* does the browser support box-shadow CSS3 */ if (head.features.boxShadow) { // do stuff } /* does the browser support box-shadow CSS3 */ .box-shadow-true { box-shadow: 3px 3px 3px black; } /* Add a custom feature yourself */ head.feature(member, true); /* Now use it */ if (head.features.member) { // do stuff } .member-true { display: block; } {% endhighlight %}
